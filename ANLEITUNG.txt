API erreichbar unter 
	http://127.0.0.1:8000
oder	http://localhost:8000/

mit dem /generate-token Endpoint könnt ihr ein TOKEN generieren, den dann einfügen wenn ihr zB /evaluate aufrufen wollt, müsst ihr das so eingeben: http://127.0.0.1:8000/evaluate?token=eyJhbGciOiJIUzI1NiIsInR5c.... (hier den ganzen Token einfügen) 
	oder wenn ihr über /docs testet dann ins TOKEN Feld einfügen!

1. Generiert euch eine Datei die ".env" heißt im Hauptverzeichnis und füllt sie hiermit aus:

SECRET_KEY=
PYTHONPATH=.
MONGO_URI = 

Den secret key müsst ihr euch jeweils selber generieren zB mit so einem script:
	
	import secrets

	new_secret_key = secrets.token_hex(32)
	print(new_secret_key)

den Output dann als SECRET_KEY in eurer .env Datei abspeichern. 

PYTHONPATH sollte so passen mit dem . und die MOONGO_URI ist so ein String: mongodb+srv://<individual_username>:<password>@cluster.mongodb.net/lfb
einfach <individual_username> durch euren login namen ersetzen und <password> selbstverständlich mit eurem passwort. (Die Daten hierfür sende ich euch im Slack)

2. Auf https://cloud.mongodb.com/v2/66e40dcc395bf74c2a73b977#/security/network/accessList oder auf mongodb.com und dann Network Access (unten links) klicken
	dort dann aus +ADD CURRENT IP ADRESS klicken (jedes mal neu wenn ich keine statische IP habt) 

3. der server wird gestartet aus dem root folder mit dem Kommando: $ uvicorn app.main:app --reload 
	(nicht vergessen das lfb_env conda-environment zu starten vorher!)
	Falls ihr das environment noch nicht habt: im root folder das terminal eurer Wahl öffnen und das hier eingeben:
	
	conda env create -f environment.yml
	conda activate lfb_env

4. Die Tests können so alle durchlaufen (geht natürlich auch einzeln sonst) $ pytest tests/
	aktuell funktionieren nur 4 von 6 tests da müssen wir nochmal ran..

Das sind unsere very basic ersten Endpoints: 

/predict: For making predictions.
/evaluate: For model evaluation.
/database/add: For adding data.
/generate-token: For generating JWT tokens
